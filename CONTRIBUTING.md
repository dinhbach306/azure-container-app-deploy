# Contributing to Azure Container Apps Deploy Action

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
- A clear description of the problem
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Your environment (OS, Azure CLI version, etc.)

### Suggesting Enhancements

We welcome suggestions for new features or improvements:
- Open an issue describing the enhancement
- Explain why this enhancement would be useful
- Provide examples of how it would be used

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes (`git commit -m 'Add some amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

### Development Guidelines

- Follow existing code style and conventions
- Add comments for complex logic
- Update documentation for any changes
- Test your changes with real Azure deployments if possible
- Keep commits focused and atomic

### Testing

Before submitting a pull request:
1. Test the action with a real Azure Container App deployment
2. Verify all inputs work as expected
3. Check that error handling works correctly
4. Ensure documentation is up to date

## Code of Conduct

### Our Pledge

We pledge to make participation in our project a harassment-free experience for everyone.

### Our Standards

- Using welcoming and inclusive language
- Being respectful of differing viewpoints
- Gracefully accepting constructive criticism
- Focusing on what is best for the community
